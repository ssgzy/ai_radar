# 问题与坑点

关联：[[当前状态]] | [[任务笔记 - AI Radar 搭建]] | [[版本迭代记录]] | [[下一步]]

## 当前已知问题

### 1. 现有目录里有早期脚本和历史输出

- 影响：如果不先检查，很容易把历史残留误当成正式结构
- 当前处理：先读后改，在现有 `ai_radar` 上规范化重构，不新建第二套目录

### 2. 当前目录不是 Git 仓库

- 影响：无法直接用 Git 跟踪本轮差异
- 当前处理：本轮先以 Obsidian 笔记和运行日志维护差异记录

### 3. 本地 Ollama 状态未知

- 影响：V1 是否能做真实中文总结取决于本机是否已启动 Ollama 且已有模型
- 当前处理：V1 开发前会单独检测 Ollama 服务和模型列表

### 4. 现有 summarizer 设计过于写死

- 影响：模型、超时、提示词和输出格式难以扩展
- 当前处理：V1 重构时会把模型调用封装到 `utils/ollama_client.py`

### 5. `qwen3.5:9b` 在当前机器上首轮响应明显偏慢

- 影响：即使是很短的测试提示，也可能等待很久，影响开发和调试体验
- 当前处理：实测后将默认模型切换为 `mistral:7b`，同时保留 `qwen3.5:9b` 和 `deepseek-r1:8b` 作为可切换模型

### 6. 模型输出偶尔会带 Markdown 粗体噪音

- 影响：Obsidian 笔记会出现 `**` 等不必要装饰，影响阅读
- 当前处理：在 prompt 中明确禁止装饰符号，并在解析阶段增加清洗逻辑

### 7. 当前评分仍是启发式规则，不是最终版智能排序

- 影响：对于复杂主题或弱信号内容，分数可能仍偏粗糙
- 当前处理：V2 先用可解释的规则评分，保证稳定和可追踪；V3/V5 再考虑更强的模型辅助评分
