## V1：最小可用版本已完成

### 本版目标

- 跑通 `arXiv` collector
- 跑通 `GitHub` collector
- 跑通本地 Ollama 中文总结
- 输出 `processed JSON`
- 输出 Obsidian 条目笔记和日报
- 保证运行中有明确进度和等待提示

### 本版新增内容

- 完整的项目目录骨架
- 配置层：`sources.yaml`、`models.yaml`、`prompts.yaml`、`obsidian.yaml`
- 数据模型层：`RawItem`、`ProcessedItem`、`SourceRunRecord`
- 采集层：`ArxivCollector`、`GitHubCollector`
- 处理层：`LocalOllamaSummarizer`
- 导出层：`JsonExporter`、`ObsidianExporter`
- 运行入口：`run_once.py`、`run_manual.py`、`run_ai_radar.py`
- 状态落盘：`last_run.json`、`seen_ids.json`、`seen_urls.json`、`seen_titles.json`、`source_checkpoints.json`
- 调试输出：prompt / response 文件
- 基础测试：`test_summarizer.py`、`test_pipeline.py`

### 与上一版差异

- V0 只有核查和文档初始化；V1 已经具备真实可运行的业务闭环
- 新增了模块化模型策略，而不是把模型写死在单脚本中
- 新增了 Obsidian 导出和日报，而不只是 JSON
- 新增了等待心跳提示，长时间模型调用时不会完全安静

### 已解决问题

- 解决了旧脚本结构过散、难以扩展的问题
- 解决了运行结果只落原始 JSON、不落 Obsidian 的问题
- 解决了模型等待时终端几乎无反馈的问题
- 解决了 GitHub 无描述时总结质量明显偏弱的问题

### 仍存在的问题

- 还没有正式的 scoring / tags / dedupe
- 还没有 RSS / Hacker News / News
- 还没有真正的 `launchd` / `cron` 调度脚本
- `qwen3.5:9b` 在当前机器上延迟偏高，不适合作为默认模型

### 下一版计划

### 与 V2 的差异

- V2 只支持 `arXiv` 与 `GitHub`；V3 已扩展到 `RSS`、`Hacker News`、`News`
- V2 只有 `merged/scored JSON`；V3 新增 `deduped JSON` 和 `duplicate report`
- V2 的条目笔记主要是论文和项目；V3 新增统一的资讯类 `inbox` 笔记
- V2 的日报按分数排序；V3 的日报会同时展示去重前后数量和每来源去重后统计

### 已解决问题

- 解决了系统只能看论文和 GitHub 的问题
- 解决了多来源合并后没有正式去重环节的问题
- 解决了资讯类内容没有统一落盘位置的问题
- 解决了 `Hacker News` 中短关键词 `ai` 误判无关条目的问题
- 解决了 `deduped/scored JSON` 中 `note_path` 回写为空的问题

### 仍存在的问题

- 当前去重仍是启发式规则，不是 embedding 或模型辅助版本
- 最新基线样本中重复数为 `0`，还没有拿到一个能明显展示合并去重效果的真实案例
- `RSS / Hacker News / News` 的相关性过滤仍比较轻，需要后续继续增强
- 还没有接入失败条目归档和重试机制
