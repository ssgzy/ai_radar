# 系统架构

关联：[[项目总览]] | [[模块说明]] | [[任务笔记 - AI Radar 搭建]] | [[版本迭代记录]] | [[Collector 设计]] | [[Agent 设计]] | [[Exporter 设计]] | [[自动运行方案]]

## 分层逻辑

### 1. 数据采集层

- `collectors/` 负责从不同来源拉取内容
- 当前已实现 `arXiv` 和 `GitHub`
- 后续扩展 `RSS`、`Hacker News`、`Reddit`、`Product Hunt`、`News`
- 细化说明见 [[Collector 设计]]

### 2. 内容处理层

- `agents/` 负责总结、评分、打标签、去重和日报生成
- 当前已实现本地 Ollama 摘要、评分和标签
- V3 再加入 merge / dedupe 的正式能力
- 细化说明见 [[Agent 设计]]、[[评分与标签设计]] 与 [[Prompt 设计]]

### 3. 输出层

- `processed JSON` 用于机器处理和回溯
- `scored JSON` 用于排序和筛选
- `Markdown` 和 `Obsidian` 用于人工阅读和知识沉淀
- 后续加入日报、周报和 dashboard note
- 细化说明见 [[Exporter 设计]] 与 [[数据说明]]

### 4. 自动化层

- `run_once.py` 用于单次执行
- `run_manual.py` 用于手动调试
- `run_ai_radar.py` 作为统一入口
- `schedulers/` 为 `cron` 和 `launchd` 预留
- 自动运行限制和现状见 [[自动运行方案]]

### 5. 知识管理层

- `obsidian/` 维护中文项目知识库
- `docs/` 维护工程说明文档
- 所有关键结论都要能从 [[项目总览]] 和 [[结果索引]] 找到
