# 评分与标签设计

关联：[[Agent 设计]] | [[版本迭代记录]] | [[问题与坑点]] | [[下一步]]

## V2 当前实现

### 标签

- 使用 `KeywordTagger`
- 根据 `config/scoring.yaml` 中的 `tagging.topics` 和 `source_tags` 生成标签
- 当前标签以“可解释、可维护”为优先

### 评分

- 使用 `HeuristicScorer`
- 当前分成三个维度：
  - 新颖性
  - 落地信号
  - 个人相关性
- 使用权重合成总分
- 再映射到推荐级别：
  - 高优先级跟进
  - 重点关注
  - 快速浏览
  - 低优先级

## 当前取舍

- 先不把评分完全交给模型
- 原因是启发式规则更稳定、可追踪、可解释
- 后续如果要升级，可以引入模型辅助评分，但仍建议保留规则层兜底
